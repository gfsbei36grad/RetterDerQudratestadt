Spiel.java

Die Klasse Spiel managet den Spielablauf. Sie erzeugt bsp. Items und Bomben und zeigt Texte an.

Vererbung

Spiel erbt von der Greenfoot-Klasse World, hat also die Eigenschaften, welche die World-Klasse mitliefert. 

World
  ↑
Spiel

Attribute

Attribut Datentyp Beschreibung
spielerStartX int X-Startposition des Spielers
spielerStartY int Y-Startposition des Spielers
spieler Spieler Referenz auf den Spieler
pausiert boolean Speichert, ob das Spiel pausiert ist
liegtPowerUp boolean Gibt an, ob bereits ein PowerUp existiert
gameOver boolean Gibt an, ob das Spiel läuft oder vorbei ist

Konstruktor

Methode Beschreibung
Spiel() Erstellt die Welt + Methode actorsLaden() wird aufgerufen.

Methoden

Methode Beschreibung
act() Greenfoot-Methode
actorsLaden() Dauerhaft bleibende Actors werden zur Welt hinzugefügt
itemSpawner() wird aufgerufen durch act(), spawnt zufällig Items
bombenSpawner() wird aufgerufen durch act(), spawnt zufällig Bomben. Je mehr Punkte desto höher die Spawnrate
gameOver() Beendet das Spiel und zeigt den Game-Over-Bildschirm an.
setLiegtPowerUp(boolean wert) Setter Methode für das Attribut "liegtPowerUp"
getPausiert() Getter Methode; Gibt zurück, ob das Spiel pausiert ist.
getLiegtPowerUp() Getter Methode; Gibt zurück, ob aktuell ein PowerUp existiert.
getSpieler() Gibt die Referenz auf den Spieler zurück.
text() Wird aufgerufen durch act(), aktualisiert den Inhalt der Texte, die im Spiel angezeigt werden

Beziehungen zu anderen Klassen

Klasse Beziehung
Spieler Wird in dieser Klasse instanziiert
Bombe Wird zufällig erzeugt
Item Wird zufällig erzeugt
Strassenbahn Wird in dieser Klasse instanziiert
GameOverScreen Der Game Over Screen wird bei Spielende instanziiert
