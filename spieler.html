Spieler.java

Die Klasse Spieler managet den Spieler. Sie verarbeitet die Steuerung, das Einsammeln von Items und PowerUps und verwaltet Punkte sowie Leben.

Vererbung

Spieler erbt von der Greenfoot-Klasse Actor, hat also die Eigenschaften, welche die Actor-Klasse mitliefert. 

Actor
  ↑
Spieler

Attribute

Attribut Datentyp Beschreibung
geschwindigkeit int Bewegungsgeschwindigkeit des Spielers
punkte int Aktuelle Punktzahl des Spielers
leben int Aktuelle Anzahl an Leben
powerUpZeit double Zählt die Dauer eines aktiven PowerUps
wechselAnimation double Wird für den Wechsel der Spieleranimation verwendet
spiel Spiel Referenz auf die Spielwelt
bewegungLinks boolean Speichert, ob sich der Spieler nach links bewegt
bewegungRechts boolean Speichert, ob sich der Spieler nach rechts bewegt
bewegungUnten boolean Speichert, ob sich der Spieler nach unten bewegt
powerUpAktiv boolean Gibt an, ob aktuell ein PowerUp aktiv ist
neuerHighScoreErreicht boolean Gibt an, ob ein neuer Highscore erreicht wurde
powerUpText String Speichert die aktuelle Fähigkeit des aktiven PowerUps in diesem Attribut, damit die Klasse Spiel es oben links als Text anzeigen kann

Methoden

Methode Beschreibung
addedToWorld(World welt) Greenfoot-Methode
act() Greenfoot-Methode
highscore() Prüft, ob ein neuer Highscore erreicht wurde und speichert diesen
animation() Wechselt zwischen zwei Spielerbildern (idle Animation)
einsammeln() Prüft, ob ein Item eingesammelt wird
powerUpReset() Setzt ein aktives PowerUp nach einer bestimmten Zeit zurück

schiesse() Instanziiert eine neue Kugel
getPunkte() Getter Methode für das Attribut "punkte"
minusLeben() Zieht dem Spieler ein Leben ab, kann die "gameOver" Methode der Klasse Spiel aufrufen
getLeben() Getter Methode für das Attribut "leben"
getPowerUpText() Getter Methode für das Attribut "powerUpText"
neuerHighScore() Getter Methode für das Attribut "neuerHighScoreErreicht" (true/false)
